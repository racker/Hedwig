Hedwig is a component library for graphing monitoring data for Rackspace.

It is based on web components, so it can fit inside of any web UI application.



How to get started

To get started simply install Hedwig into your application


Installing Hedwig

Install using npm 


npm install rackspace/hedwig


How To Use

Hedwig uses web components so it is quite easy to use.

Just add a component tag to your page and add any data-* attributes for configuration 


<hedwig-graph data-type="MAAS_cpu" data-field="usage_average" data-height="200" data-width="400" data='[someBoundData]' ></hedwig-graph>

This will render the graph, taking in the data and mapping according to the components perscribed data format.


data-type
This is required for the graph to render, the value must correspond with the list of accepted measurement names.

data-field
This is required for the graph to render, the value must correspond with the matching field name for the measurement.



Grouping & MultiLine Graphs

One of the features Hedwig has is the ability to create multiline graphs. This can be relevant for when more than one device is wanting to be displayed in a graph or more than one monitor for a specific device.

For displaying a multiline graph be sure to specify the key in the JSON data that will be used to group the metrics. Consider the following data:

            "mean": 0.99,
            "time": "2018-11-24T18:58:21Z",
            "device": "6221-forestofdean.prod.com"
            },
            {
            "mean": 0.67,
            "time": "2018-11-25T23:58:21Z",
            "device": "6221-forestofdean.prod.com"
            },
            {
            "mean": 0.33,
            "time": "2018-11-26T02:58:21Z",
            "device": "6221-forestofdean.prod.com"
            },
            {
            "mean": 0.45,
            "time": "2018-11-24T18:58:21Z",
            "device": "7877-ministryofmagic.staging.com"
            },
            {
            "mean": 0.10,
            "time": "2018-11-25T23:58:21Z",
            "device": "7877-ministryofmagic.staging.com"
            },
            {
            "mean": 0.89,
            "time": "2018-11-26T02:58:21Z",
            "device": "7877-ministryofmagic.staging.com"
            }]

If you're wanting to graph this data in a multiline format where device is the grouping qualifier, you'd then add the data-group attribute and enter the value of the key being used for grouping. 

<hedwig-graph data-type="MAAS_cpu" data-field="usage_average" data-group="device" data-height="200" data-width="400" data='[someBoundData]' ></hedwig-graph>

This will render the graph, with multiline capabilities
